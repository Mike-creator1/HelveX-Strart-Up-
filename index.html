API

Three lines.
First call.

The same primitives our team uses internally. Typed, streaming, well-behaved under load. No magic — just an API that respects your time.

app.ts · @helvex/sdk 
// Stream a response from Ether — sub-100ms first token
import { helvex } from "@helvex/sdk";

const stream = await helvex.complete({
  model: "ether-1",
  stream: true,
  messages: [
    { role: "system", content: "You are a precise editor." },
    { role: "user",  content: "Tighten this paragraph." },
  ],
});

for await (const chunk of stream) process.stdout.write(chunk.delta);

FAQ

Specific questions.
Specific answers.

Can the router pick a model automatically?

Yes — pass "model": "auto" and we route per request based on your latency, cost, and quality targets. You can also constrain the choice (e.g. "prefer": ["nexus","prometheus"]) or pin a model explicitly. Routing decisions are exposed in the response so you can audit them.

Do you train on customer inputs?

Never. Inputs and outputs through the API and Studio are not used for training. Enterprise plans add zero-retention mode (immediate deletion after the response is delivered) and a signed DPA.

Where is data processed?

Default residency is US-East. EU residency (Frankfurt + Zurich) is available on Pro and Enterprise. Choose at the workspace level. Cross-region routing is opt-in, never automatic.

How does Prometheus stay coherent over many hours?

Structured planning, plan critique, and a memory hierarchy with verifiable checkpoints. Agents can be paused, inspected, resumed, or replayed. We wrote up the engineering details in the post linked under Research.

Can we self-host or run inside our cloud account?

Yes — Enterprise supports private deployments inside your AWS, GCP, or Azure account, with VPC peering and customer-managed keys. Air-gapped is available on case-by-case review. Talk to sales for the architecture conversation.

What's the migration path from another provider?

Most teams swap their base URL and headers, run their existing eval suite against our models, and ship. Our SDK ships shims for the common provider shapes so you can A/B test without rewriting prompts. We've migrated teams of 1,000+ engineers in under a sprint.
